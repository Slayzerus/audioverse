Example 2: Using in Focusable
```typescript
import { navigationLogger } from '../../services/navigationLogger';

useEffect(() => {
  if (!ref.current) {
    navigationLogger.error('FOCUSABLE_REF_NULL_ON_MOUNT', { id });
    return;
  }
  
  try {
    register({ id, ref, ...layout, isDropdown });
    navigationLogger.debug('FOCUSABLE_REGISTERED', { id });
  } catch (error) {
    navigationLogger.error('FOCUSABLE_REGISTRATION_FAILED', {
      id,
      error: error.message
    });
  }
}, [id, layout.x, layout.y, layout.width, layout.height, isDropdown]);
```

💡 BEST PRACTICES

1. Use Correct Log Levels:
   ├─ DEBUG: Development-only detailed info
   ├─ INFO: Important navigation events (rare)
   ├─ WARN: Recoverable issues (common)
   └─ ERROR: Critical failures (very rare)

2. Include Context in Data:
   ├─ Element IDs
   ├─ Page pathname
   ├─ Position/dimension info
   ├─ Input information
   └─ Element counts

3. Recovery Strategies:
   ├─ Log the error
   ├─ Attempt recovery
   ├─ Fallback gracefully
   └─ Log fallback action

4. Performance Considerations:
   ├─ Keep in-memory log size bounded (1000)
   ├─ Debug logging only in development
   ├─ Batch analytics/error tracking calls
   └─ Don't log every frame
