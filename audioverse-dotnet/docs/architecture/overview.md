## Wzorce architektoniczne

### Clean Architecture
- **Domain** — encje, enumy, interfejsy repozytoriów
- **Application** — logika biznesowa, CQRS, serwisy
- **Infrastructure** — implementacje repozytoriów, zewn?trzne API
- **API** — kontrolery, middleware, konfiguracja

### CQRS (Command Query Responsibility Segregation)
- **Commands** — operacje modyfikuj?ce (Create, Update, Delete)
- **Queries** — operacje odczytuj?ce
- **Handlers** — MediatR do obs?ugi polece? i zapyta?

### Repository Pattern
- Interfejsy w **Domain**
- Implementacje EF Core w **Infrastructure**
- Dapper dla read-heavy operacji

## Konfiguracja zale?no?ci

### Program.cs

```csharp
// Rejestracja warstw
builder.Services.AddInfrastructure(builder.Configuration);
builder.Services.AddApplication();

// MediatR
builder.Services.AddMediatR(cfg => 
    cfg.RegisterServicesFromAssembly(typeof(Application.DependencyInjection).Assembly));

// FluentValidation
builder.Services.AddValidatorsFromAssembly(typeof(Application.DependencyInjection).Assembly);

// SignalR
builder.Services.AddSignalR();
```

### DependencyInjection.cs (Infrastructure)

```csharp
public static IServiceCollection AddInfrastructure(this IServiceCollection services, IConfiguration config)
{
    // Database
    services.AddDbContext<AudioVerseDbContext>(options =>
        options.UseNpgsql(config.GetConnectionString("DefaultConnection")));

    // Repositories
    services.AddScoped<IKaraokeRepository, KaraokeRepositoryEF>();
    services.AddScoped<IEventRepository, EventRepositoryEF>();
    services.AddScoped<IUserProfileRepository, UserProfileRepositoryEF>();
    // ... inne repozytoria

    // External APIs
    services.AddHttpClient<IBggClient, BggClient>();
    services.AddHttpClient<ISteamClient, SteamClient>();

    return services;
}
```
